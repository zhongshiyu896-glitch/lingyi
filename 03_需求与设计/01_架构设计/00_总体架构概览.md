## 第一章：系统边界定义

### 1.1 ERPNext 负责的具体功能

ERPNext 是 headless 后端和权威 ERP 引擎，负责标准单据、财务、库存、权限和审批。ERPNext 表位于 PostgreSQL `public` schema。

| ERPNext DocType / 能力 | 负责范围 | 说明 |
| --- | --- | --- |
| `Item` | 款式资料、面辅料、半成品、成品物料 | 款式主档不在 FastAPI 重建事实表，FastAPI 只扩展 BOM 和服装业务属性 |
| `Sales Order` | 销售订单 | 订单事实源，生产计划按已审批 SO 生成 |
| `Purchase Order` | 采购订单 | 标准采购事实源，BOM 展开后的采购需求最终进入采购链路 |
| `Delivery Note` | 发货单 | 出库发货事实源 |
| `Stock Entry` | 发料、收料、调拨、生产入库 | 外发发料、外发回料、生产入库均通过 Stock Entry 落账 |
| `Stock Ledger Entry` | 实时库存台账 | 库存余额和库存成本权威来源，FastAPI 不直接写入 |
| `GL Entry` / `GL Ledger` | 总账流水 | 财务事实源，FastAPI 不直接写入 |
| `Payment Entry` | 收付款 | 应收应付结算能力 |
| `Account` | 会计科目 | 财务核算基础 |
| `Workflow` | 审批流 | 所有审批回归 ERPNext Workflow，不继续扩展废弃的自建审批流 |
| `Role` / `User Permission` | 角色和数据权限 | 权限控制的权威来源 |
| `Work Order` | 标准生产工单 | 生产计划集成时由 FastAPI 调用 ERPNext 创建 |
| `Job Card` | 工序卡 | 与工票系统关联，用于工序执行闭环 |

ERPNext 不负责服装行业特有的 BOM 展开算法、外发加工过程管理、工票日薪计算、工价档案、款式利润计算和加工厂对账明细。

### 1.2 FastAPI 自建负责的具体模块

FastAPI + SQLAlchemy + Pydantic 是自建服装业务层。自建表统一放在 PostgreSQL `ly_schema` 下。

| 模块 | 自建对象 | 负责范围 |
| --- | --- | --- |
| BOM 管理 | `ly_apparel_bom`、`ly_apparel_bom_item`、`ly_bom_operation` | 款式 BOM、尺码配比、损耗用量、工序定义、外发工序标识 |
| 外发加工管理 | `ly_subcontract_order`、`ly_subcontract_material`、`ly_subcontract_receipt` | 外发单、发料、回料、验货、扣款、状态机 |
| 工票/车间管理 | `ys_workshop_ticket`、`ys_workshop_daily_wage` | 工票登记、撤销、批量操作、日薪统计、Job Card 关联 |
| 工价档案 | `ly_operation_wage_rate` | 本厂工序计件价、外发单件价、生效版本 |
| 款式利润计算 | `ly_style_profit_snapshot`、`ly_style_profit_detail` | 标准成本、实际成本、利润和毛利率计算 |
| 加工厂对账单 | `ly_factory_statement`、`ly_factory_statement_item` | 按加工厂和期间生成对账单、扣款、实付金额 |

FastAPI 负责业务编排和算法，不负责替代 ERPNext 标准单据 CRUD，不直接写 ERPNext 库存台账和财务账本。

### 1.3 Vue3 前端负责的具体模块

Vue3 + Element Plus + TypeScript 是业务用户操作入口，按衣算云 UI 风格实现页面。

| 前端模块 | 页面范围 | 调用对象 |
| --- | --- | --- |
| BOM 管理 | 款式 BOM 列表、BOM 编辑、BOM 展开预览 | FastAPI `/api/bom/` |
| 外发加工 | 外发单列表、发料、回料、验货、扣款 | FastAPI `/api/subcontract/` |
| 工票/车间 | 工票登记、撤销、批量导入、日薪统计 | FastAPI `/api/workshop/` |
| 生产计划 | SO 转生产计划、Work Order 同步、Job Card 状态 | FastAPI `/api/production/` |
| 款式利润报表 | 款式利润、订单利润、标准实际对比 | FastAPI `/api/reports/style-profit/` |
| 加工厂对账 | 对账单列表、明细、打印、确认 | FastAPI `/api/factory-statements/` |
| 销售/库存入口 | SO、库存台账、Stock Entry 查询入口 | 优先通过 FastAPI 聚合，必要时跳转 ERPNext |
| 权限和首页 | 菜单、按钮、首页指标、角色控制 | FastAPI 聚合 ERPNext 权限结果 |

Vue3 不直连 PostgreSQL，不直接调用 ERPNext 写接口。除后台管理跳转外，业务页面统一调用 FastAPI。

### 1.4 ERPNext 和 FastAPI 的数据流向

| 业务动作 | 写入 ERPNext public schema | 写入 FastAPI ly_schema | 数据流向 |
| --- | --- | --- | --- |
| 维护款式资料 | `Item` | 不重复建款式事实表 | Vue3 -> FastAPI -> ERPNext `Item` |
| 维护 BOM | 不写 ERPNext 标准 BOM 作为首要事实源 | `ly_apparel_bom`、`ly_apparel_bom_item`、`ly_bom_operation` | Vue3 -> FastAPI -> `ly_schema` |
| BOM 展开采购需求 | 可后续生成 `Purchase Order` 或采购申请 | 记录展开结果或快照 | FastAPI 计算 -> ERPNext 采购链路 |
| 外发发料 | `Stock Entry`、`Stock Ledger Entry` | `ly_subcontract_order`、`ly_subcontract_material` | FastAPI 建业务单 -> ERPNext 落库存 |
| 外发回料 | `Stock Entry`、`Stock Ledger Entry` | `ly_subcontract_receipt` | FastAPI 验收 -> ERPNext 入库 |
| 工票登记 | 可关联 `Job Card` | `ys_workshop_ticket` | Vue3/PDA -> FastAPI -> `ly_schema` |
| 日薪计算 | 不写 ERPNext 财务账本 | `ys_workshop_daily_wage` | FastAPI 计算并留痕 |
| Work Order 创建 | `Work Order`、`Job Card` | `ly_production_plan`、`ly_work_order_link` | FastAPI 生成计划 -> ERPNext 创建工单 |
| 款式利润计算 | 读取 `Sales Order`、`Purchase Receipt`、`Stock Ledger Entry` | `ly_style_profit_snapshot`、`ly_style_profit_detail` | ERPNext 事实 + FastAPI 成本 -> 利润快照 |
| 加工厂对账 | 可后续生成 `Purchase Invoice` 草稿 | `ly_factory_statement`、`ly_factory_statement_item` | FastAPI 汇总确认 -> ERPNext 应付建议 |

## 第三章：数据库设计规范

### 3.1 ly_schema 命名规范

1. FastAPI 自建表统一放在 PostgreSQL `ly_schema`。
2. ERPNext 表统一在 PostgreSQL `public` schema，FastAPI 不创建同名替代表。
3. 跨 schema 引用不建立数据库物理外键，使用业务字段保存 ERPNext DocType 和 name。
4. 所有自建表必须包含审计字段：`created_at`、`created_by`、`updated_at`、`updated_by`。
5. 所有写入类业务表必须包含 `status`，金额或库存相关表必须包含 `company`。

### 3.2 表命名规范

| 表类型 | 命名规范 | 示例 |
| --- | --- | --- |
| BOM 主表 | `ly_apparel_bom` | `ly_schema.ly_apparel_bom` |
| BOM 明细 | `ly_apparel_bom_item` | `ly_schema.ly_apparel_bom_item` |
| BOM 工序 | `ly_bom_operation` | `ly_schema.ly_bom_operation` |
| 外发主表 | `ly_subcontract_order` | `ly_schema.ly_subcontract_order` |
| 外发明细 | `ly_subcontract_material`、`ly_subcontract_receipt` | `ly_schema.ly_subcontract_receipt` |
| 工票 | `ys_workshop_ticket` | `ly_schema.ys_workshop_ticket` |
| 工价 | `ly_operation_wage_rate` | `ly_schema.ly_operation_wage_rate` |
| 利润 | `ly_style_profit_snapshot`、`ly_style_profit_detail` | `ly_schema.ly_style_profit_snapshot` |
| 对账 | `ly_factory_statement`、`ly_factory_statement_item` | `ly_schema.ly_factory_statement` |

### 3.3 字段命名规范

| 字段类别 | 命名规范 | 示例 |
| --- | --- | --- |
| 主键 | `id` | UUID 或 bigint，由工程实现阶段确定 |
| ERPNext 引用 | `erpnext_doctype`、`erpnext_name` | `erpnext_doctype='Work Order'` |
| Item 引用 | `item_code` | 对应 ERPNext `Item.name` |
| Supplier 引用 | `supplier` | 对应 ERPNext `Supplier.name` |
| Sales Order 引用 | `sales_order` | 对应 ERPNext `Sales Order.name` |
| Work Order 引用 | `work_order` | 对应 ERPNext `Work Order.name` |
| 数量 | `*_qty` | `planned_qty`、`accepted_qty` |
| 单价 | `*_rate` | `wage_rate`、`subcontract_rate` |
| 金额 | `*_amount` | `deduction_amount`、`net_amount` |
| 比例 | `*_rate` | `loss_rate`、`rejected_rate` |
| 状态 | `status` | `draft`、`processing`、`completed` |
| 日期 | `*_date` | `posting_date`、`from_date`、`to_date` |

### 3.4 索引设计原则

1. 每张表必须有主键索引。
2. 业务编号必须有唯一索引，例如 `bom_no`、`subcontract_no`、`statement_no`。
3. 高频筛选字段必须建索引，例如 `company`、`status`、`item_code`、`supplier`、`sales_order`、`work_order`。
4. 明细表必须对主表外键字段建索引，例如 `bom_id`、`subcontract_order_id`、`statement_id`。
5. 报表表必须对统计维度建组合索引，例如 `(company, item_code, from_date, to_date)`。
6. 幂等业务必须建立唯一索引，例如工票 `ticket_key + process_name + color + size + operation_type + work_date`。
7. ERPNext 引用字段建议建立组合索引 `(erpnext_doctype, erpnext_name)`。

```json

  "detail": {

  }
```

### 4.5 API 设计原则

1. 创建使用 `POST`，查询使用 `GET`，更新使用 `PUT/PATCH`，取消或提交使用动作接口。
2. 所有写接口必须支持幂等控制。
3. 前端只依赖 FastAPI 契约，不直接依赖 ERPNext 内部接口。
4. 接口路径使用小写英文和连字符。
5. 错误码按模块前缀区分，例如 `BOM_`、`SUBCONTRACT_`、`WORKSHOP_`、`PRODUCTION_`。

## 第五章：与 ERPNext 的集成方案

### 5.1 ERPNext REST API 接入方式

1. FastAPI 通过 ERPNext REST API 调用标准 DocType。
2. 标准读取使用 `/api/resource/{DocType}`。
3. 标准提交、取消、特殊动作使用 ERPNext 暴露的 method API 或受控后端方法。
4. FastAPI 使用服务账号或 token 鉴权访问 ERPNext，禁止 Vue3 直接持有 ERPNext 写权限 token。
5. FastAPI 记录每次 ERPNext 写入的请求参数、返回结果、业务单号和错误信息。

### 5.2 共享 PostgreSQL 的连接管理

1. ERPNext 和 FastAPI 共用同一个 PostgreSQL 实例。
2. ERPNext 表位于 `public` schema，FastAPI 自建表位于 `ly_schema`。
3. FastAPI 使用 SQLAlchemy 管理连接池，只写 `ly_schema`。
4. FastAPI 可以只读查询 ERPNext `public` schema 的必要数据，但优先通过 ERPNext API 获取带权限语义的数据。
5. FastAPI 不直接更新 `public` schema 下的 ERPNext 标准表。
6. 涉及 ERPNext 标准单据创建、提交、取消的操作，必须通过 ERPNext API 完成。

### 5.3 ERPNext 自定义字段命名规范

1. ERPNext 自定义字段统一使用 `custom_ly_*` 前缀。
2. 字段名使用小写英文和下划线，例如 `custom_ly_style_code`。
3. 自定义字段只补充服装业务维度，不改变 ERPNext 标准字段语义。
4. 自定义字段需要在接口规范或数据字典中登记用途、类型、所属 DocType。
5. 已废弃或历史字段不得继续扩展新逻辑，需在审计记录中标记。

## 第六章：前端架构概览

### 6.1 前端页面分层

Vue3 前端目录按以下层次组织：

| 层级 | 目录 | 用途 |
| --- | --- | --- |
| 页面层 | `/06_前端/lingyi-pc/src/views/` | 业务页面入口 |
| 组件层 | `/06_前端/lingyi-pc/src/components/` | 可复用表格、表单、弹窗、状态标签 |
| 状态层 | `/06_前端/lingyi-pc/src/stores/` | 用户、权限、菜单、页面缓存 |
| API 层 | `/06_前端/lingyi-pc/src/api/` | FastAPI 请求封装 |
| 路由层 | `/06_前端/lingyi-pc/src/router/` | 页面路由和权限守卫 |
| 类型层 | `/06_前端/lingyi-pc/src/types/` | TypeScript 类型定义 |

### 6.2 与 FastAPI 后端的接口调用方式

1. 前端统一通过 `/src/api/*.ts` 调用 FastAPI。
2. API 层统一处理 `{code, message, data}` 响应格式。
3. 分页组件统一消费 `{items, total, page, page_size}`。
4. 错误提示统一使用 FastAPI 返回的 `message`，字段级错误读取 `detail`。
5. 前端按钮权限由 FastAPI 返回的权限动作控制，不在页面硬编码角色。

### 6.3 与 ERPNext API 的调用方式

1. 常规业务页面不直接调用 ERPNext API。
2. 需要打开 ERPNext 标准单据时，前端只跳转到 ERPNext 对应页面。
3. 需要读取 ERPNext 数据时，前端调用 FastAPI 聚合接口，由 FastAPI 再访问 ERPNext。
4. 管理员后台场景可以保留 ERPNext Desk 入口，但不作为衣算云风格业务页面的主路径。
