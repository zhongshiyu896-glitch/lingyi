## 1. 业务目标

款式利润报表用于按款式、销售订单和时间范围分析利润，输出标准成本、实际成本、差异、利润金额和利润率。该模块依赖 ERPNext 销售与库存事实，也依赖 FastAPI 自建的 BOM、外发、工票和生产计划事实。

## 2. 系统边界

### 2.1 ERPNext 负责

| ERPNext 能力 | 用途 | TASK-005 使用方式 |
| --- | --- | --- |
| `Sales Order` | 销售订单和订单金额候选来源 | 读取订单、客户、公司、订单行、销售单价、数量 |
| `Sales Invoice` | 实际确认收入候选来源 | TASK-005A 必须评估是否优先于 Sales Order |
| `Purchase Receipt` | 采购实际入库成本候选来源 | 读取采购入库金额和物料成本 |
| `Stock Ledger Entry` | 库存实际成本候选来源 | 读取库存价值变动和实际成本 |
| `Item` | 款式和物料主数据 | 校验 item、物料名称、单位、公司范围 |
| `Role / User Permission` | 权限事实源 | 权限源不可用时 fail closed |

### 2.2 FastAPI 自建负责

| 自建模块 | 用途 |
| --- | --- |
| BOM 管理 | 标准材料用量、损耗、标准工序成本 |
| 外发加工管理 | 外发加工费、扣款、结算锁定事实 |
| 工票/车间管理 | 实际工票数量、撤销数量、计件工资 |
| 生产计划集成 | 销售订单、BOM、Work Order、Job Card 映射关系 |
| 款式利润报表 | 利润快照、明细、标准实际对比、报表查询 |

### 2.3 前端负责

1. 款式利润列表页。
2. 利润快照生成入口。
3. 利润明细钻取页。
4. 标准成本和实际成本对比展示。
5. 筛选项：公司、日期、款式、销售订单、客户。
6. 导出、打印和权限按钮联动。

## 3. 数据流候选

1. ERPNext `Sales Order / Sales Invoice` 提供收入候选事实。
2. BOM 展开结果提供标准材料成本和标准工序成本。
3. ERPNext `Purchase Receipt / Stock Ledger Entry` 提供实际材料成本候选事实。
4. 工票模块提供实际工票成本。
5. 外发模块提供加工费、扣款和结算事实。
6. 生产计划模块提供 SO、BOM、Work Order、Job Card 的关联桥。
7. TASK-005 最终生成不可变利润快照和利润明细。

## 6. 利润口径待决策项

1. 收入口径：使用 `Sales Order` 订单金额，还是 `Sales Invoice` 已确认收入。
2. 实际材料成本口径：使用 `Purchase Receipt`，还是 `Stock Ledger Entry`。
3. 外发成本口径：使用验货事实，还是对账锁定事实。
4. 工票成本口径：是否以撤销后净数量为准。
5. 历史快照策略：重算必须新建快照，不覆盖旧快照。
6. 多公司权限：是否以 `company + item_code + sales_order` 三维资源权限控制。
7. 费用分摊：制造费用、管理费用、其他费用是否进入 V1。
