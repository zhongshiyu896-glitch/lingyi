## 1. 设计目标与冻结范围

冻结移动端与小程序渠道边界，明确渠道承载范围、鉴权与会话约束、只读/候选写入边界、权限审计要求、前端门禁与禁止事项。

本任务只输出设计冻结，不写业务代码，不做移动端/小程序实现，不做真实联调，不做生产发布。

## 2. 渠道边界冻结

### 2.1 纳入设计范围

| 渠道能力 | 冻结范围 | 当前阶段结论 |
|---|---|---|
| 登录与会话 | 统一走领意现有鉴权入口、会话续期、退出登录、设备标识管理 | 仅冻结边界，不实现 |
| 待办与消息触达 | 承载已审批通过模块的待办、消息、提醒、只读状态回看 | 仅冻结只读语义 |
| 轻量看板 | 承载生产、成本、供应链等已冻结设计模块的只读投影 | 仅允许只读投影 |
| 协同确认候选入口 | 对供应链协同中的确认、回执、签收类动作仅冻结候选入口语义 | 写入默认冻结 |
| 扫码 / 上传 / 离线缓存 | 只定义通道能力边界、审计与安全约束 | 不进入实现 |
| 推送与通知绑定 | 只定义 token 绑定、解绑、消息确认语义 | 不进入实现 |

### 2.2 不纳入本任务范围

1. 不新增移动端独立业务域。
2. 不新增 ERPNext 直写接口。
3. 不开放库存、财务、生产、采购的移动端写链路。
4. 不定义 024B / 024C / 024D 的实现细节。
5. 不定义 App Store / 小程序平台发布流程。

## 3. 业务承载边界

### 3.1 承载原则

1. 移动端与小程序只是现有业务能力的渠道层，不得形成旁路业务逻辑。
2. 仅允许承载已完成设计冻结的模块能力投影：`TASK-021A`、`TASK-022A`、`TASK-023A`。
3. 未经单独设计审计通过的写操作，一律保持冻结。
4. 所有候选写入必须在后续实现任务中继续遵守 `Adapter -> Outbox -> Worker` 链路，不得在本任务中放开。

### 3.2 对外语义

1. 资源级访问必须先做动作权限，再做资源过滤。
2. 对外语义继续冻结为：`资源存在但越权` 与 `资源不存在` 统一返回 `not-found` 形态。
3. 内部必须保留真实拒绝原因并写安全审计，对外不得暴露可区分信息。

## 4. 鉴权、会话与设备边界

1. 移动端 / 小程序不得新增独立权限体系，必须复用现有账号、角色、资源权限基线。
2. 会话建立、续期、注销、设备绑定、消息 token 绑定必须走领意受控 API，不得由终端直接访问 ERPNext / Frappe。
3. 终端本地只允许缓存非敏感只读投影；`Authorization`、`Cookie`、`Token`、`Secret`、明文 DSN 不得写入本地持久化或日志。
4. 设备维度只可作为附加风控与审计维度，不得替代用户与资源权限判断。
5. 小程序侧不得暴露服务端密钥、ERPNext 凭据、Outbox 内部事件键。

## 5. 数据与集成边界

1. 所有终端请求统一进入领意受控 API 层，不得直连 ERPNext `/api/resource`。
2. 读取 ERPNext / 外部系统事实时，统一走 `TASK-008 Fail-Closed Adapter`。
3. 后续若存在终端触发的候选写入，统一走 `TASK-009 Outbox`，由受控 worker 异步执行；在 `TASK-024A` 阶段默认冻结。
4. 离线缓存、前台草稿、通知已读回执等仅可定义为候选语义，不得在本任务中放行真实写链路。
5. 外部回调、通知回执、消息签名校验必须保留 fail-closed 语义，签名失败、结构异常、来源不可信均按拒绝处理并记审计。

## 6. 权限与资源维度

### 6.1 权限原则

1. 移动端与小程序不新增“万能移动权限”。
2. 终端能力必须复用所属业务模块动作权限，例如生产、成本、供应链等模块动作权限。
3. 渠道层只允许增加受控的终端场景约束，不得降低既有权限门槛。

### 6.2 最小资源维度

移动端 / 小程序侧最小资源维度冻结为：

- `company`
- `user_id` / `employee_id`
- `supplier_id`
- `warehouse`
- `work_order`
- `cost_center`
- `request_id`
- `device_id`
- `message_id`

资源字段缺失默认拒绝（fail closed）。

## 7. 审计要求（继承 TASK-007）

### 7.1 安全审计事件

- 401 未认证
- 403 禁止
- 资源越权
- device bind rejected
- session refresh rejected
- request_id rejected
- client version blocked
- callback signature invalid
- permission source unavailable

### 7.2 操作审计事件

- login
- logout
- refresh
- dashboard_read
- todo_read
- message_ack
- scan_request
- upload_request
- sync_queued
- sync_completed
- sync_failed

### 7.3 脱敏要求

日志与审计中禁止记录：

- Authorization
- Cookie
- Token
- Secret
- password
- 明文 DSN
- 小程序密钥 / 推送密钥原文

## 8. 前端与终端门禁（继承 TASK-010）

2. 禁止裸 `fetch/axios` 绕过 API client。
3. 禁止 runtime codegen、来自不可信源的 dynamic import、未审计 worker。
4. 禁止在客户端持久化敏感凭据。
5. 禁止在 `TASK-024A` 未通过前推进 `TASK-024B ~ TASK-024D` 实现。
6. 终端上传、扫码、推送绑定如涉及写入，只能在后续实现任务中单独放行。
