# TASK-S-002 本地验收回归执行手册

## 1. 目标

在不修改主线业务代码的前提下，执行本地验收/冒烟回归，并输出结构化报告。

## 2. 前置条件

1. 已进入目录：`07_后端/lingyi_service`
2. 可访问目标 API 地址（`--base-url`）
3. 按需提供 token（`--token`）

## 3. 执行命令

### 3.1 单模块执行（BOM）

```bash
bash scripts/run_acceptance_smoke.sh \
  --module bom \
  --base-url http://127.0.0.1:8000 \
  --token "<runtime-token>" \
  --report-dir ../../03_需求与设计/06_交付与验收/TASK-S-002/sample_reports
```

### 3.2 全模块执行（13 个关键接口）

```bash
bash scripts/run_acceptance_smoke.sh \
  --module all \
  --base-url http://127.0.0.1:8000 \
  --token "<runtime-token>" \
  --report-dir ../../03_需求与设计/06_交付与验收/TASK-S-002/sample_reports
```

### 3.3 指定失败报告目录（便于留痕）

```bash
bash scripts/run_acceptance_smoke.sh \
  --module all \
  --base-url http://127.0.0.1:18081 \
  --token "<runtime-token>" \
  --report-dir ../../03_需求与设计/06_交付与验收/TASK-S-002/failure_sample
```

### 3.4 使用配置文件

```bash
bash scripts/run_acceptance_smoke.sh \
  --config tools/acceptance_smoke/config.example.json
```

## 4. 输出结果

执行后将生成：

- `report.json`
- `report.md`

报告包含：

- 顶层：`task_id`、`module`、`total_cases`、`passed`、`failed`、`skipped`、`elapsed_ms`
- 明细：每个 case 的 `endpoint/method/expected_status/actual_status/expected_code/actual_code/pass/error`
- `report.md`：模块通过率、失败明细、跳过原因、耗时统计、执行命令和报告生成时间

## 5. 退出码

- `0`：全部通过
- `1`：存在失败、跳过、连接失败、配置错误或环境阻塞

## 6. 审计关注点

1. 是否按 `--module` 选择正确 case 文件。
2. 是否按顺序执行并记录每条 case 结果。
3. 是否输出 `report.json` 与 `report.md`。
4. 是否包含 BOM/Workshop/Production/Subcontract 四模块 13 个关键接口。
5. 是否保持边界：不改主线业务代码、不触及 TASK-005/TASK-006。
