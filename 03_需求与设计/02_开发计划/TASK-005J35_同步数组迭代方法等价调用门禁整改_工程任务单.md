## 一、任务目标

修复同步数组迭代方法 `.call/.apply/Reflect.apply` 等价调用未进入 TASK-005J34 method descriptor 路径的问题。J34 已修复直调 `reduce/reduceRight/flatMap/findIndex/findLast/findLastIndex`，但 `Array.prototype.reduce.call(...)`、`Array.prototype.reduce.apply(...)`、`Reflect.apply(Array.prototype.reduce, ...)` 等等价调用仍可同步执行 callback、污染 Worker 参数数组并绕过门禁。

本任务要求建立统一 `resolveRuntimeArrayIterationCallDescriptor()` 或等价机制，将 direct call、`.call`、`.apply`、`Reflect.apply` 统一还原为同一结构：`{ methodName, iterableExpression, callbackExpression, initialValueExpression, argumentMode }`，然后复用 J34 的 method descriptor map 和 current item 参数位规则。

本任务只修前端只读契约门禁，不开放款式利润快照创建入口，不进入 TASK-006。

## 二、背景问题

审计意见书第 153 份确认 J34 直调同步迭代方法已通过，但以下等价调用仍返回通过：

```ts
const args = [new URL('./readonly-worker.ts', import.meta.url)]

Array.prototype.reduce.call(
  [[args]],
  (acc, [alias]) => {
    alias[0] = 'data:text/javascript,postMessage(1)'
    return acc
  },
  0,
)

new Worker(...args)
```

```ts
const args = [new URL('./readonly-worker.ts', import.meta.url)]

Reflect.apply(Array.prototype.reduce, [[args]], [
  (acc, [alias]) => {
    alias[0] = 'data:text/javascript,postMessage(1)'
    return acc
  },
  0,
])

new Worker(...args)
```

根因：当前 `applyRuntimeIterationCallbackSummaryAtCall()` 只识别 callee 自身是迭代方法成员。表达式为 `Array.prototype.reduce.call(...)` 时，当前成员名是 `call`，不是 `reduce`，因此不会进入 J34 的 descriptor map。

## 四、实现要求

### 4.1 必须建立统一数组迭代调用描述符

必须新增或改造为统一调用描述符解析逻辑，至少输出以下字段：

| 字段 | 含义 |
| --- | --- |
| `methodName` | 同步数组迭代方法名，如 `reduce`、`findIndex`、`flatMap` |
| `iterableExpression` | 被迭代对象，也就是 direct call 的 receiver 或 `.call/.apply/Reflect.apply` 的 thisArg |
| `callbackExpression` | 同步执行的 callback |
| `initialValueExpression` | `reduce/reduceRight` 的 initialValue，可为空 |
| `argumentMode` | `direct` / `call` / `apply` / `reflect_apply` |

要求：

1. direct call、`.call`、`.apply`、`Reflect.apply` 必须全部走同一 descriptor。
2. descriptor 必须复用 J34 的 method descriptor map。
3. descriptor 必须复用 J34 的 current item 参数位规则。
4. 无法静态还原 `methodName`、`iterableExpression`、`callbackExpression` 或参数数组时，只要作用域内存在 tracked Worker args，必须 fail closed。
5. 不允许为 `.call/.apply/Reflect.apply` 单独复制一套污染逻辑。

### 4.2 必须覆盖 `.call` 等价调用

必须拦截：

```ts
const args = [new URL('./readonly-worker.ts', import.meta.url)]

Array.prototype.reduce.call(
  [[args]],
  (acc, [alias]) => {
    alias[0] = 'data:text/javascript,postMessage(1)'
    return acc
  },
  0,
)

new Worker(...args)
```

要求：

1. `Array.prototype.reduce.call(thisArg, callback, initialValue)` 必须还原为 `methodName=reduce`。
2. 第 1 个实参 `thisArg` 必须作为 iterable。
3. 第 2 个实参必须作为 callback。
4. 第 3 个实参必须作为 initialValue。
5. `reduce` 仍必须按 callback 第 2 个参数作为 current item。
6. `findIndex/flatMap/findLast/findLastIndex/reduceRight` 的 `.call` 同口径。

### 4.3 必须覆盖 `.apply` 等价调用

必须拦截：

```ts
const args = [new URL('./readonly-worker.ts', import.meta.url)]

Array.prototype.reduce.apply([[args]], [
  (acc, [alias]) => {
    alias[0] = 'data:text/javascript,postMessage(1)'
    return acc
  },
  0,
])

new Worker(...args)
```

要求：

1. `Array.prototype.reduce.apply(thisArg, argsArray)` 必须还原为 `methodName=reduce`。
2. `thisArg` 必须作为 iterable。
3. `argsArray[0]` 必须作为 callback。
4. `argsArray[1]` 必须作为 initialValue。
5. `argsArray` 无法静态展开时必须 fail closed。
6. `.apply` 的参数数组别名、数组污染、元素重写必须复用既有数组状态图；无法证明安全时 fail closed。

### 4.4 必须覆盖 `Reflect.apply` 等价调用

必须拦截：

```ts
const args = [new URL('./readonly-worker.ts', import.meta.url)]

Reflect.apply(Array.prototype.reduce, [[args]], [
  (acc, [alias]) => {
    alias[0] = 'data:text/javascript,postMessage(1)'
    return acc
  },
  0,
])

new Worker(...args)
```

要求：

1. `Reflect.apply(target, thisArg, argsArray)` 必须解析 target 是否为同步数组迭代方法。
2. `target=Array.prototype.reduce` 时，必须还原为 `methodName=reduce`。
3. `thisArg` 必须作为 iterable。
4. `argsArray[0]` 必须作为 callback。
5. `argsArray[1]` 必须作为 initialValue。
6. `target`、`thisArg` 或 `argsArray` 无法静态还原时必须 fail closed。
7. `Reflect.apply` 别名如果已有通用 call descriptor 能识别，必须接入；无法识别时 fail closed。

### 4.5 必须支持常见方法源表达式

至少支持以下 method source：

```ts
Array.prototype.reduce
Array.prototype['reduce']
Array.prototype.flatMap
Array.prototype.findIndex
Array.prototype.findLast
Array.prototype.findLastIndex
Array.prototype.reduceRight
```

要求：

1. 字面量 bracket key 必须归一为方法名。
2. 非字面量 method key 无法证明安全时 fail closed。
3. `const reduce = Array.prototype.reduce; reduce.call(...)` 如果已有 alias 体系可识别，必须接入；如果无法静态证明 target 安全，必须 fail closed。
4. `const AP = Array.prototype; AP.reduce.call(...)` 如果已有 namespace alias 体系可识别，必须接入；如果无法静态证明 target 安全，必须 fail closed。

### 4.6 必须保留 J34 直调路径

不得回退以下已通过路径：

1. `[[args]].reduce((acc, [alias]) => ..., 0)`
2. `[[args]].reduceRight((acc, [alias]) => ..., 0)`
3. `[[args]].flatMap(([alias]) => ...)`
4. `[[args]].findIndex(([alias]) => ...)`
5. `[[args]].findLast(([alias]) => ...)`
6. `[[args]].findLastIndex(([alias]) => ...)`
7. 未知 collection 的 direct call fail closed。
8. clean readonly direct call 不被误杀。
