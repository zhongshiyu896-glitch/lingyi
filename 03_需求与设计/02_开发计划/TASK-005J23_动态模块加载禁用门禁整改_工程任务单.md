## 一、任务目标

修复 `check-style-profit-contracts.mjs` 对动态模块加载执行内联代码的高危绕过，禁止 style-profit surface 内通过 `import('data:...')`、变量动态 import、Blob URL、`URL.createObjectURL(new Blob(...))` 等方式加载运行时代码。只允许可静态证明安全的本地静态动态 import，例如 Vue 路由懒加载本地 `.vue/.ts` 文件。

## 二、背景问题

TASK-005J22 已关闭第 140 份审计中的 timer `.call/.apply/Reflect.apply` 字符串参数绕过，但审计官在第 141 份审计中复现以下动态模块加载绕过：

```ts
import('data:text/javascript,export default Object.assign')
import('data:text/javascript,export default Reflect.set')

const url = 'data:text/javascript,export default Object.assign'
import(url)

```

这些路径可以绕过静态门禁，在运行时加载并返回受禁 mutator。款式利润前端当前为只读页面，不需要从 data/blob/变量 URL 加载运行时代码，因此必须 fail closed。

## 四、实现要求

### 4.1 禁用 data/blob/http/javascript 动态 import

必须识别并拦截以下形式：

```ts
import('data:text/javascript,export default Object.assign')
import('data:text/javascript,export default Reflect.set')
import('blob:https://example.local/module')
import('javascript:alert(1)')
import('https://cdn.example.com/mod.js')
```

要求：

1. `ImportExpression` 的 source 为字符串字面量时必须检查协议。
2. `data:`、`blob:`、`javascript:`、`http:`、`https:` 等协议必须 fail closed。
3. 模板字面量或字符串拼接可静态折叠时，折叠后命中上述协议必须 fail closed。
4. 含表达式模板、变量拼接、函数返回值等无法静态证明为本地安全路径时，必须 fail closed。

### 4.2 变量动态 import 必须 fail closed

必须识别并拦截以下形式：

```ts
const url = 'data:text/javascript,export default Object.assign'
import(url)

const safeMaybe = getModuleUrl()
import(safeMaybe)

import(moduleMap[key])
```

要求：

1. `import(variable)` 在 style-profit surface 内默认 fail closed。
2. 只有 source 可静态解析为安全本地字面量路径时才允许通过。
3. 对象/数组索引、函数返回、条件表达式、模板表达式无法证明安全时 fail closed。
4. `import(/* @vite-ignore */ url)` 不得豁免，仍 fail closed。

### 4.3 Blob URL module loading 必须 fail closed

必须识别并拦截以下形式：

```ts
const blobUrl = URL.createObjectURL(new Blob(['export default Object.assign'], { type: 'text/javascript' }))
import(blobUrl)

import(URL.createObjectURL(new Blob(['export default Reflect.set'], { type: 'text/javascript' })))
```

要求：

1. `URL.createObjectURL(new Blob(...))` 在 style-profit surface 内必须 fail closed。
2. `window.URL.createObjectURL`、`globalThis.URL.createObjectURL` 必须等价处理。
3. `Blob` 内容无需解析；只要用于生成 module URL 或可被 import 的 URL 即 fail closed。
4. `new Blob(..., { type: 'text/javascript' })`、`application/javascript`、`text/ecmascript` 等脚本 MIME 必须 fail closed。
5. 包装、变量中转、函数返回、IIFE、条件表达式中的 Blob URL 生成必须 fail closed。

### 4.4 允许的安全动态 import

允许以下静态本地懒加载：

```ts
import('@/views/style_profit/StyleProfitList.vue')
import('@/views/style_profit/StyleProfitDetail.vue')
import('./local-readonly-helper')
import('../readonly/formatter')
```

要求：

1. 仅允许字符串字面量，且路径必须是本地相对路径或 `@/` 项目内路径。
2. 不允许协议 URL。
3. 不允许变量、表达式、模板表达式。
4. 不允许路径中出现 `..` 后逃出项目根的风险路径。
5. 路径目标如果位于 style-profit surface，仍必须接受既有只读契约扫描。

### 4.5 Worker / Script 动态代码加载同步收口

如果门禁已扫描到以下高危入口，也必须 fail closed：

```ts
new Worker(URL.createObjectURL(new Blob(['...'])))
const script = document.createElement('script')
script.src = URL.createObjectURL(new Blob(['...']))
```

要求：

1. 不要求完整实现 DOM 污染追踪，但直接 Blob URL 代码加载入口必须 fail closed。
2. 不得误杀普通静态本地图片/样式资源 URL。
