## 一、任务目标

修复 `check-style-profit-contracts.mjs` 对运行时代码生成入口的高危绕过，禁止 style-profit surface 内使用 `eval`、`Function`、`new Function`、`globalThis/window.Function` 及其别名、间接调用、动态成员访问。任何可以在运行时生成或取回受禁 mutator 的代码生成入口，都必须 fail closed。

## 二、背景问题

TASK-005J19 已关闭第 137 份审计中的动态成员名、`Reflect.get`、optional chain 等源引用隐藏绕过，但审计官在第 138 份审计中确认仍可通过运行时代码生成绕过静态门禁：

```ts
Function('return Object.assign')()(item, { onClick: openHelp })
new Function('return Object.defineProperty')()(item, 'onClick', { value: openHelp })
eval('Reflect.set')(item, 'onClick', openHelp)
globalThis.Function('return Object.assign')()(item, { onClick: openHelp })
```

运行时代码生成会让门禁无法可靠静态分析，且款式利润前端当前为只读页面，不需要任何 `eval/Function` 能力。因此本任务要求在 style-profit surface 内禁用运行时代码生成入口。

## 四、实现要求

### 4.1 禁用 eval

必须识别并拦截以下形式：

```ts
eval('Reflect.set')
window.eval('Object.assign')
globalThis.eval('Object.defineProperty')
;(0, eval)('Object.assign')
```

要求：

1. 直接 `eval(...)` 必须 fail closed。
2. `window.eval/globalThis.eval` 必须 fail closed。
3. 间接 eval，例如 `(0, eval)(...)` 必须 fail closed。
4. `eval?.(...)` optional call 必须 fail closed。
5. 不需要解析 eval 字符串内容；出现 eval 即 fail closed。

### 4.2 禁用 Function 构造器

必须识别并拦截以下形式：

```ts
Function('return Object.assign')
new Function('return Object.defineProperty')
globalThis.Function('return Object.assign')
window.Function('return Reflect.set')
```

要求：

1. `Function(...)` 必须 fail closed。
2. `new Function(...)` 必须 fail closed。
3. `globalThis.Function/window.Function` 必须 fail closed。
4. `Function?.(...)`、`globalThis?.Function?.(...)` optional chain 必须 fail closed。
5. 不需要解析 Function 字符串内容；出现 Function 构造器即 fail closed。

### 4.3 禁用 Function/eval 别名

必须识别并拦截以下形式：

```ts
const make = Function
make('return Object.assign')

const Eval = eval
Eval('Reflect.set')

const { Function: Fn } = globalThis
Fn('return Object.assign')

const { eval: run } = window
run('Object.defineProperty')
```

要求：

1. 直接别名必须 fail closed。
2. 解构别名、解构重命名必须 fail closed。
3. 赋值式解构别名必须 fail closed。
4. 命名空间别名下的 `Function/eval` 必须 fail closed。
5. 别名来源不明、被重写或无法静态确认时，在 style-profit surface 内疑似代码生成调用必须 fail closed。

### 4.4 禁用动态成员访问入口

必须识别并拦截以下形式：

```ts
globalThis['Function']('return Object.assign')
window['eval']('Object.assign')
globalThis['Func' + 'tion']('return Reflect.set')
window[`ev${'al'}`]('Object.defineProperty')
```

要求：

1. 对 `Function/eval` 的字符串字面量、模板字面量、字符串拼接成员名做安全折叠。
2. 折叠结果命中 `Function/eval` 时 fail closed。
3. `globalThis/window` 动态成员 key 无法静态证明安全时，在 style-profit surface 内 fail closed。

### 4.5 禁用代码生成源引用进入容器或返回值

以下形式即使未立即调用，也必须 fail closed：

```ts
const dangerous = Function
const dangerous = eval
const holder = { make: Function }
const holder = [eval]
const getFn = () => Function
const maker = condition ? Function : Function
```

要求：

1. `Function/eval` 源引用放入变量、数组、对象、函数返回、IIFE、条件表达式时 fail closed。
2. 容器中包含 `Function/eval` 源引用，即使未调用也 fail closed。
3. 包装容器，如 `Object.freeze({ make: Function })`，必须 fail closed。

## 五、必须新增反向测试

至少新增以下反向测试，每条都必须明确断言门禁失败：

1. `Function('return Object.assign')()(item, { onClick: openHelp })`
2. `new Function('return Object.defineProperty')()(item, 'onClick', { value: openHelp })`
3. `eval('Reflect.set')(item, 'onClick', openHelp)`
4. `globalThis.Function('return Object.assign')()(item, { onClick: openHelp })`
5. `window.Function('return Reflect.set')()(item, 'onClick', openHelp)`
6. `globalThis.eval('Object.assign')(item, { onClick: openHelp })`
7. `window.eval('Object.defineProperty')(item, 'onClick', { value: openHelp })`
8. `;(0, eval)('Object.assign')(item, { onClick: openHelp })`
9. `const make = Function; make('return Object.assign')()`
10. `const run = eval; run('Reflect.set')`
11. `const { Function: Fn } = globalThis; Fn('return Object.assign')()`
12. `const { eval: run } = window; run('Object.defineProperty')`
13. `globalThis['Function']('return Object.assign')()`
14. `window['eval']('Object.assign')`
15. `globalThis['Func' + 'tion']('return Reflect.set')()`
16. `` window[`ev${'al'}`]('Object.defineProperty') ``
17. 源引用存在但未调用也失败：`const dangerous = Function`
18. 容器存在但未调用也失败：`const holder = { make: Function }`
19. 函数返回源引用失败：`const getFn = () => Function`
20. 条件表达式源引用失败：`const maker = condition ? Function : Function`
