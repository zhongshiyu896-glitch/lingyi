## 2. 实现说明

### 2.1 统一迭代调用 descriptor
在 `check-style-profit-contracts.mjs` 新增/改造：
- `resolveRuntimeArrayIterationCallDescriptor(callNode, calleeExpression)`
- descriptor 字段：
  - `methodName`
  - `iterableExpression`
  - `callbackExpression`
  - `initialValueExpression`
  - `argumentMode`（`direct/call/apply/reflect_apply`）

统一覆盖：
- 直调：`iterable.method(callback, ...)`
- 等价调用：`Array.prototype.xxx.call(...)`
- 等价调用：`Array.prototype.xxx.apply(...)`
- 等价调用：`Reflect.apply(Array.prototype.xxx, thisArg, argsArray)`

### 2.2 J34 method descriptor map 复用
- 所有路径统一复用 `runtimeArrayIterationMethodDescriptorMap`。
- `reduce/reduceRight` 继续使用 callback 第 2 参数作为 current item。
- `flatMap/findIndex/findLast/findLastIndex` 继续使用 callback 第 1 参数作为 current item。

### 2.3 参数归一与 fail-closed
- 统一复用 `resolveRuntimeInvocationArgumentsFromNodes`、`resolveRuntimeArgumentArrayElements`。
- `.apply/Reflect.apply` 参数数组不可静态展开时：`unresolvedArguments` -> fail closed。
- method target 不可静态确认（例如 `reduce.call(...)` alias 形态）时：`unresolvedTarget` -> fail closed。

### 2.4 Reflect.apply 安全旁路修正
- 新增静态识别 `Array.prototype.<iterationMethod>` 的辅助函数。
- 在 `resolveRuntimeCallDescriptor` 中，对 `Reflect.apply(Array.prototype.iterationMethod, ...)` 不再误判为 mutator unresolved target。
- 在 `collectRuntimeSourceReferenceFindings` 中新增 safe iteration Reflect.apply 旁路，避免把合法迭代调用误判为 mutator source。

### 2.5 迭代 summary 生效后避免参数逃逸误判
- 在 `visit(CallExpression)` 中，新增 `appliedRuntimeIterationSummary`。
- 当迭代 summary 已应用时，不再走通用 `CallArgumentEscape`，避免合法迭代参数把 tracked array 误标 `escaped`。
