# Sprint3 候选任务清单

- 输出时间：2026-04-16

## REL-004 平台 CI / Required Check 闭环

- 任务编号：REL-004
- 任务名称：平台 CI / required check 闭环
- 任务目标：完成 GitHub hosted runner、artifact、required check 的平台级闭环证据，打通“本地封版 -> 平台门禁”链路。
- 前置依赖：Sprint2 本地封版复盘完成；仓库远端策略确认。
- 允许范围：CI workflow、required check 配置、平台证据文档、任务日志。
- 禁止范围：业务功能改造、ERPNext 业务写入、跨模块需求并行混入。
- 审计前置要求：
  1. 明确平台执行人和权限边界。
  2. 明确 required check 名称与分支保护策略。
  3. 提供 run URL、artifact、结论截图/日志。
- 验收标准：
  1. required check 在平台真实生效。
  2. 关键 workflow 可复跑并产出 artifact。
  3. 审计记录可回溯到具体 run 与提交 SHA。

## REL-005 ERPNext 生产联调

- 任务编号：REL-005
- 任务名称：ERPNext 生产联调与权限源复验
- 任务目标：验证生产环境下 fail-closed 行为、权限源可用性、主数据一致性与回退策略。
- 前置依赖：REL-004 通过；生产联调窗口与账号审批完成。
- 允许范围：只读联调、受控写动作演练（按任务单放行）、联调证据文档。
- 禁止范围：无审计开放写入口、绕过 outbox、绕过权限源、临时硬编码放行。
- 审计前置要求：
  1. 明确联调对象（Supplier/Item/Account/Cost Center/docstatus）。
  2. 明确故障演练（timeout/5xx/401/403/malformed）计划。
  3. 明确敏感信息脱敏策略。
- 验收标准：
  1. fail-closed 在生产联调环境可复验。
  2. 权限源异常不出现 fail-open。
  3. 联调证据包含请求链路、错误码、审计日志。

## TASK-013 质量管理增强

- 任务编号：TASK-013
- 任务名称：质量管理增强（阶段二）
- 任务目标：在 TASK-012 基线上扩展质量统计、规则配置和分析能力（不触发财务写入）。
- 前置依赖：TASK-012 本地封版通过；REL-005 至少完成只读联调验证。
- 允许范围：质量统计增强、查询性能优化、规则配置与报表能力、测试补强。
- 禁止范围：自动扣款、自动返工工单、自动报废入账、财务记账链路。
- 审计前置要求：
  1. 状态机和关键口径不回退。
  2. 新增字段/索引/迁移策略先冻结。
  3. 写入口权限动作与资源字段先冻结。
- 验收标准：
  1. 统计口径可复验且与基线一致。
  2. 关键性能指标达标。
  3. 不突破 TASK-012 已冻结业务边界。

## TASK-014 销售库存联动增强

- 任务编号：TASK-014
- 任务名称：销售库存联动增强
- 任务目标：在 TASK-011 只读基线上扩展联动查询、导出与分析视图，为后续受控写动作做准备。
- 前置依赖：TASK-011 本地封版通过；REL-005 只读联调完成。
- 允许范围：只读聚合、缓存优化、导出增强、权限过滤增强。
- 禁止范围：直接写 ERPNext、库存写入、销售单据提交、outbox 新增写动作。
- 审计前置要求：
  1. 继续保证 GET-only 边界。
  2. 继续保证防枚举策略。
  3. 继续保证 front contract 负向用例覆盖。
- 验收标准：
  1. 接口仍保持只读。
  2. Customer/warehouse/item 过滤无越权。
  3. 前后端契约与场景数门禁不回退。

## TASK-015 Outbox 模板回迁

- 任务编号：TASK-015
- 任务名称：Outbox 模板回迁到历史模块
- 任务目标：将 TASK-009 通用模板回迁到 TASK-002/003/004/006，收敛 event_key/claim/lease/retry 容差差异。
- 前置依赖：TASK-009 本地封版通过；相关模块基线可回归。
- 允许范围：outbox service/worker 最小改造、模板接入、回归测试扩充。
- 禁止范围：跨模块业务改口径、无证据改状态机、直接删旧逻辑无兼容层。
- 审计前置要求：
  1. 每个模块回迁前后状态机矩阵对照。
  2. 并发语义（PostgreSQL）验证计划。
  3. 失败重试和审计策略保持一致。
- 验收标准：
  1. event_key 稳定且禁入字段一致。
  2. claim_due 原子条件一致。
  3. 历史模块回归测试通过且无行为回退。

## TASK-016 权限矩阵生产化

- 任务编号：TASK-016
- 任务名称：权限矩阵生产化与服务账号最小权限落地
- 任务目标：把 TASK-007 权限与资源规范映射到生产角色矩阵并完成最小权限策略。
- 前置依赖：REL-005 完成权限源生产可用性验证。
- 允许范围：权限配置、角色矩阵文档、校验脚本、审计监控补强。
- 禁止范围：临时硬编码全放行、跳过资源权限、关闭审计。
- 审计前置要求：
  1. 动作权限矩阵完整。
  2. 资源字段映射完整。
  3. 权限源不可用降级策略明确（仍 fail-closed）。
- 验收标准：
  1. 关键角色最小权限通过复核。
  2. 服务账号动作边界可审计。
  3. 越权路径均有安全审计证据。

## TASK-017 前端门禁 AST 深化

- 任务编号：TASK-017
- 任务名称：前端门禁 AST 深化与性能优化
- 任务目标：提升 TASK-010 engine 的 AST 语义分析覆盖与执行性能，降低误报和漏报。
- 前置依赖：TASK-010 本地封版通过；现有模块 contract 运行稳定。
- 允许范围：engine 规则增强、fixture 扩展、报告能力增强、性能优化。
- 禁止范围：放宽 fail-closed 默认策略、删除历史高危反向用例、移除旧模块门禁入口。
- 审计前置要求：
  1. 新规则需提供正反 fixture。
  2. 旧模块场景数不得回退。
  3. 误报与漏报样例需有基准对照。
- 验收标准：
  1. frontend-contract-engine 场景数与覆盖率提升。
  2. style-profit/factory-statement/sales-inventory/quality 场景不回退。
  3. verify 时长与稳定性满足门禁要求。

## 结语

Sprint3 候选任务建议按“平台闭环 -> 联调验证 -> 基座回迁 -> 业务增强”顺序推进。每个任务均需单独下发任务单并先审计通过后再进入实现。
