## 四、实现要求

### 4.1 Object 解构别名必须识别

必须识别以下形式，并映射回原始 API：

```ts
const { defineProperty } = Object
const { defineProperties } = Object
const { assign } = Object
```

调用以下别名时，必须按 `Object.defineProperty/Object.defineProperties/Object.assign` 原规则拦截：

```ts
defineProperty(item, 'onClick', { value: openHelp })
defineProperties(item, { onClick: { value: openHelp } })
assign(item, { onClick: openHelp })
```

### 4.2 Object 解构重命名必须识别

必须识别以下形式：

```ts
const { defineProperty: dp } = Object
const { assign: merge } = Object
```

调用 `dp(...)`、`merge(...)` 时，必须按原 API 等价拦截。

### 4.3 Reflect 解构别名必须识别

必须识别以下形式：

```ts
const { set } = Reflect
const { set: reflectSet } = Reflect
```

调用 `set(item, 'onClick', openHelp)` 或 `reflectSet(item, 'onClick', openHelp)` 时，必须按 `Reflect.set` 等价拦截。

### 4.4 命名空间别名必须识别

必须识别以下形式：

```ts
const Obj = Object
const R = Reflect
```

并拦截：

```ts
Obj.defineProperty(item, 'onClick', { value: openHelp })
Obj['assign'](item, { onClick: openHelp })
R.set(item, 'onClick', openHelp)
R['set'](item, 'onClick', openHelp)
```

### 4.5 别名安全边界

1. 别名分析只要求覆盖同一文件内的 `const/let/var` 声明。
2. 如果别名被重新赋值、来源不明或与局部函数冲突，遇到 style-profit surface 内疑似 action 注入时必须 fail closed。
3. 不需要跨文件追踪 import/export。
4. 不得通过变量名白名单放行。
5. 不得缩小 TASK-005J13 已覆盖的 bracket callee、直接函数别名、变量 source/descriptor 检测范围。
