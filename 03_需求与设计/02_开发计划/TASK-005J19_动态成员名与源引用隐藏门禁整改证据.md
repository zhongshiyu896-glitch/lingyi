## 3. 实现摘要

### 3.1 动态成员名静态折叠

新增静态字符串折叠能力 `resolveStaticStringValue()`，用于处理：

- 字符串字面量
- 模板字面量（可静态求值）
- 字符串拼接（`+`）
- 数字字面量

并统一接入：

- `getStaticLiteralText()`
- `classifyPropertyNameNode()` 的 computed key 分支
- `classifyExpressionKeyNode()`

这样可以正确识别：

- `Object['define' + 'Property']`
- `Object['ass' + 'ign']`
- `Reflect['s' + 'et']`
- `` Object[`define${'Property'}`] ``

### 3.2 源引用隐藏路径收口

在 runtime mutator 源引用检测中新增 fail closed 收口：

1. `collectRuntimeMutatorSourceReferenceFindings()` 增强：
   - 继续识别直接 source 引用；
   - 新增 `RuntimeMutatorSourceUnknownNamespaceMember`；
   - 新增 `RuntimeMutatorSourceUnknownGlobalNamespaceMember`；
   - 新增 call expression 分析，支持通过调用表达式识别 source 引用。

2. `resolveRuntimeCallDescriptor()` 增强：
   - 支持 `Reflect.get` 解析结果回映到源 mutator；
   - 对已知 namespace/global container 上不可静态解析成员名时返回 `unresolvedTarget=true`，触发 fail closed。

3. 新增 `resolveRuntimeReflectGetCall()`：
   - 识别 `Reflect.get` / `Reflect['get']`；
   - 对 `(Object|Reflect, 静态 key)` 映射回 `Object.defineProperty/Object.assign/Reflect.set/Reflect.apply`；
   - key 不可静态解析时 fail closed。

### 3.3 Optional chain 与等价语法

通过 AST 路径统一处理 optional access/call 与普通 access/call 等价语义，覆盖：

- `Object.defineProperty?.(...)`
- `Object?.defineProperty?.(...)`
- `Object?.['assign']?.(...)`
- `Reflect?.['set']?.(...)`
