## 一、任务目标

修复 `check-style-profit-contracts.mjs` 中数组状态跟踪按“函数体源码位置”记录污染事件导致的绕过。必须将函数声明提升、闭包副作用、函数别名调用和无参函数调用统一按“调用点”生效：只要函数调用在 Worker/unknown constructor spread 使用点之前可能污染数组，就必须将相关数组标记为 tainted/unknown，并在 style-profit surface 内 fail closed。

本任务只修前端只读契约门禁，不开放款式利润快照创建入口，不进入 TASK-006。

## 二、背景问题

TASK-005J30 已修复直接数组污染、别名污染、`splice/unshift/push/reverse/fill` 等主路径，但审计官第 149 份审计复现函数声明提升绕过：

```ts
const args = [new URL('./readonly-worker.ts', import.meta.url), { type: 'module' }]
poison()
new Worker(...args)

function poison() {
  args[0] = 'data:text/javascript,postMessage(1)'
}
```

运行时先调用 `poison()` 污染数组，但门禁按函数体源码位置记录污染事件，误判污染发生在 `new Worker(...args)` 之后。

最高风险：当前数组状态跟踪仍是源码顺序近似，不是控制流 / 调用图语义；函数声明提升、闭包副作用、函数别名和无参函数调用都可能让运行时顺序和静态位置脱节。

## 四、实现要求

### 4.1 函数副作用必须按调用点生效

必须新增或改造 function side-effect summary，至少记录：

1. `function_id`
2. `declared_name`
3. `aliases`
4. `captures_tracked_arrays`
5. `mutates_array_ids`
6. `escapes_array_ids`
7. `has_unknown_side_effect`
8. `summary_confidence = exact | conservative | unknown`

要求：

1. 函数体内对数组的写入、mutating method、逃逸，不能按函数体源码位置直接污染使用点。
2. 必须在函数调用点应用该函数的副作用 summary。
3. 调用点位于 `new Worker(...args)` 或 `Reflect.construct(Worker, args)` 之前时，函数副作用必须影响该使用点。
4. 函数声明提升不得让函数体源码位置晚于调用点而绕过。
5. 函数 summary 无法精确计算时，在 style-profit surface 内必须 conservative fail closed。

### 4.2 函数声明提升必须识别

必须拦截以下形式：

```ts
const args = [new URL('./readonly-worker.ts', import.meta.url), { type: 'module' }]
poison()
new Worker(...args)

function poison() {
  args[0] = 'data:text/javascript,postMessage(1)'
}
```

要求：

1. FunctionDeclaration 必须在扫描调用点前纳入函数表。
2. `poison()` 调用必须在调用点污染 `args`。
3. 函数声明位于调用点之后时，仍必须按运行时提升语义处理。
4. 函数声明内通过数组别名污染，也必须污染原数组。

### 4.3 函数表达式与箭头函数副作用必须识别

必须拦截以下形式：

```ts
const poison = () => { args[0] = 'data:text/javascript,postMessage(1)' }
poison()
new Worker(...args)

const poison = function () { args.splice(0, 1, 'data:text/javascript,postMessage(1)') }
poison()
new Worker(...args)
```

要求：

1. `const/let/var poison = () => ...` 必须建立函数 summary。
2. `const/let/var poison = function () { ... }` 必须建立函数 summary。
3. 函数表达式调用点早于 spread 使用点时，必须按调用点污染数组。
4. 函数表达式被重新赋值或无法证明来源时，调用必须 conservative fail closed。

### 4.4 函数别名调用必须识别

必须拦截以下形式：

```ts
function poison() { args[0] = 'data:text/javascript,postMessage(1)' }
const run = poison
run()
new Worker(...args)
```

要求：

1. 函数别名链 `poison -> run -> exec` 必须追踪。
2. 别名调用必须应用原函数副作用 summary。
3. 别名重新赋值后必须切断关系或标记 unknown。
4. unknown function alias 被调用且存在 tracked arrays in scope 时，必须 fail closed 或将相关数组标记 unknown。

### 4.5 call/apply/bind 调用必须识别或 fail closed

必须处理以下形式：

```ts
poison.call(null)
poison.apply(null)
const bound = poison.bind(null)
bound()
```

要求：

1. `poison.call(...)`、`poison.apply(...)` 必须应用 poison 的副作用 summary。
2. `poison.bind(...)` 返回函数别名，调用 bound 时必须应用 poison 的副作用 summary。
3. 如果 callee 无法解析但疑似调用闭包函数，在 style-profit surface 内必须 fail closed。
4. 不允许 call/apply/bind 绕过数组污染。

### 4.6 闭包副作用必须识别

必须拦截以下形式：

```ts
const args = [new URL('./readonly-worker.ts', import.meta.url), { type: 'module' }]
function makePoison() {
  return function () {
    args[0] = 'data:text/javascript,postMessage(1)'
  }
}
const poison = makePoison()
poison()
new Worker(...args)
```

要求：

1. 返回函数可静态解析并捕获 tracked array 时，调用返回函数必须污染数组。
2. 返回函数不可静态解析但处于 style-profit surface 且 tracked arrays in scope 时，必须 fail closed。
3. IIFE 立即污染数组时，必须按 IIFE 调用点生效。
4. 嵌套函数如果可能污染数组且被调用，必须影响调用点之后的 spread 使用。

### 4.7 未知函数调用与数组逃逸必须保守处理

必须处理以下形式：

```ts
maybePoison()
new Worker(...args)

unknown(args)
new Worker(...args)
```

要求：

1. 未知无参函数调用如果无法证明不访问当前 scope tracked arrays，应 conservative fail closed。
2. 数组作为参数传给未知函数时，继续标记 escaped/unknown。
3. 未知函数调用出现在 spread 使用点之前，且存在 tracked arrays 用于后续 Worker/unknown constructor spread，必须 fail closed。
4. 允许明确安全函数白名单，但必须集中定义并有测试，不能用泛化白名单放行。

## 五、必须新增反向测试

至少新增以下反向测试，每条必须断言门禁失败：

1. `poison(); new Worker(...args); function poison(){ args[0] = 'data:text/javascript,postMessage(1)' }`
2. `poison(); new Worker(...args); function poison(){ args.splice(0, 1, 'data:text/javascript,postMessage(1)') }`
3. `poison(); new Worker(...args); function poison(){ const alias = args; alias[0] = 'data:text/javascript,postMessage(1)' }`
4. `const poison = () => { args[0] = 'data:text/javascript,postMessage(1)' }; poison(); new Worker(...args)`
5. `const poison = function(){ args.unshift('data:text/javascript,postMessage(1)') }; poison(); new Worker(...args)`
6. `function poison(){ args[0] = 'data:text/javascript,postMessage(1)' }; const run = poison; run(); new Worker(...args)`
7. `function poison(){ args[0] = 'data:text/javascript,postMessage(1)' }; poison.call(null); new Worker(...args)`
8. `function poison(){ args[0] = 'data:text/javascript,postMessage(1)' }; poison.apply(null); new Worker(...args)`
9. `function poison(){ args[0] = 'data:text/javascript,postMessage(1)' }; const bound = poison.bind(null); bound(); new Worker(...args)`
10. `const poison = (() => () => { args[0] = 'data:text/javascript,postMessage(1)' })(); poison(); new Worker(...args)`
11. `(() => { args[0] = 'data:text/javascript,postMessage(1)' })(); new Worker(...args)`
12. `maybePoison(); new Worker(...args)`，其中 maybePoison 无法静态证明安全且 scope 内存在 tracked args
13. `unknown(args); new Worker(...args)`
14. `poison(); Reflect.construct(Worker, args); function poison(){ args[0] = 'data:text/javascript,postMessage(1)' }`

16. `function poison(){ args[0] = 'data:text/javascript,postMessage(1)' }; const fns = [poison]; fns[0](); new Worker(...args)` 必须 fail closed 或精确拦截
