## 3. 整改实现

### 3.1 解构别名识别
在 AST 上下文分析中新增 Object/Reflect 解构别名映射：
- `const { defineProperty } = Object`
- `const { defineProperty: dp } = Object`
- `const { defineProperties } = Object`
- `const { assign } = Object`
- `const { assign: merge } = Object`
- `const { set } = Reflect`
- `const { set: reflectSet } = Reflect`

调用时自动按原始 API 归一化识别为：
- `Object.defineProperty`
- `Object.defineProperties`
- `Object.assign`
- `Reflect.set`

### 3.2 命名空间别名识别
新增 `runtimeNamespaceAliasMap`，识别：
- `const Obj = Object`
- `const R = Reflect`

并拦截等价调用：
- `Obj.defineProperty(...)`
- `Obj['assign'](...)`
- `R.set(...)`
- `R['set'](...)`

### 3.3 运行时注入等价语法统一拦截
所有等价语法最终统一进入既有运行时门禁：
- 显式 action key 注入：`style-profit forbids runtime explicit action-key injection ...`
- dynamic/unknown key 注入：`style-profit forbids runtime dynamic property injection ...`

### 3.4 既有能力保持
