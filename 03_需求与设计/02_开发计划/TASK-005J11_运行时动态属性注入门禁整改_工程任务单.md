## 1. 任务目标

修复运行时动态属性注入绕过问题。`TASK-005J10` 已将对象字面量中的 `dynamic_computed_key / unknown_key` 全域 fail closed，但仍未覆盖对象创建后的动态属性写入，例如 `item[ACTION_KEY] = goDetail`、`Object.defineProperty(item, ACTION_KEY, { value: goDetail })`、`Reflect.set(item, ACTION_KEY, goDetail)`。

本任务要求：在 style-profit surface 内，对运行时动态属性注入一律 fail closed。重点检测 AST 中的 `ElementAccessExpression` 左值赋值、`Object.defineProperty`、`Object.defineProperties`、`Reflect.set`、`Object.assign` 动态 computed source 等写入路径。读取型 `item[ACTION_KEY]` 不作为本任务阻断目标，除非它同时构成写入、定义、设置、合并或 action 入口构造。

本任务只修复前端契约门禁、测试和证据，不开放利润快照创建入口，不进入 TASK-006。

## 2. 审计问题来源

审计意见书第 129 份指出以下样例仍可绕过：

```ts
const ACTION_KEY = 'onClick'
const item = { label: '查看详情' }
item[ACTION_KEY] = goDetail
```

以及：

```ts
Object.defineProperty(item, ACTION_KEY, { value: goDetail })
```

根因：AST 门禁已覆盖对象字面量内部动态 computed key，但没有覆盖对象创建后的运行时动态属性注入。

## 5. 必做整改

### 5.1 ElementAccessExpression 左值赋值检测

在 style-profit surface 内，必须检测以下写入形式，并对非字面量 key fail closed：

1. `item[ACTION_KEY] = goDetail`
2. `item[actionMap['onClick']] = goDetail`
3. `item[getActionKey()] = goDetail`
4. `item[ACTION_KEY] ??= goDetail`
5. `item[ACTION_KEY] ||= goDetail`
6. `item[ACTION_KEY] &&= goDetail`
7. `item[ACTION_KEY] += handler`

判断口径：

1. 左侧为 `ElementAccessExpression`。
2. 操作符为赋值或复合赋值。
3. key 表达式不是字符串字面量时，必须 fail closed。
4. 不得因为目标对象名是 `item/row/config/entry` 等中性名称而放行。

### 5.2 Object.defineProperty 检测

必须检测并 fail closed：

```ts
Object.defineProperty(item, ACTION_KEY, { value: goDetail })
Object.defineProperty(item, actionMap['onClick'], { value: goDetail })
Object.defineProperty(item, getActionKey(), { value: goDetail })
```

判断口径：

1. callee 为 `Object.defineProperty`。
2. 第二个参数不是字符串字面量时，必须 fail closed。
3. 第二个参数为字符串字面量但值属于 `onClick/handler/command/onSelect/submit/execute` 时，继续按既有交互字段规则处理。

### 5.3 Object.defineProperties 检测

必须检测并 fail closed：

```ts
Object.defineProperties(item, {
  [ACTION_KEY]: { value: goDetail },
})
```

判断口径：

1. callee 为 `Object.defineProperties`。
2. 第二个参数对象中出现 dynamic / unknown computed key 时，必须 fail closed。
3. 第二个参数对象中出现字符串字面量 computed key 时，继续按既有规则处理。

### 5.4 Reflect.set 检测

必须检测并 fail closed：

```ts
Reflect.set(item, ACTION_KEY, goDetail)
Reflect.set(item, actionMap['onClick'], goDetail)
Reflect.set(item, getActionKey(), goDetail)
```

判断口径：

1. callee 为 `Reflect.set`。
2. 第二个参数不是字符串字面量时，必须 fail closed。
3. 第二个参数为字符串字面量但值属于交互字段时，继续按既有规则处理。

### 5.5 Object.assign 动态 source 检测

必须检测并 fail closed：

```ts
Object.assign(item, {
  [ACTION_KEY]: goDetail,
})
```

判断口径：

1. callee 为 `Object.assign`。
2. 任一 source object literal 中出现 dynamic / unknown computed key 时，必须 fail closed。
3. Spread、嵌套 object literal、字符串字面量 computed key 继续按 TASK-005J9/J10 规则处理。

### 5.6 读操作不作为本任务阻断目标

以下读取场景不作为本任务阻断目标，除非同时构成 action 入口或写入：

```ts
const value = item[ACTION_KEY]
if (item[ACTION_KEY]) {}
```

原因：本任务聚焦“运行时动态属性注入”。读取型动态 key 若后续被审计认定为风险，再单独出任务。
