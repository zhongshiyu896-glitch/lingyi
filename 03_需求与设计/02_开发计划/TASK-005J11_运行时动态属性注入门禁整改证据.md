## 3. 运行时动态属性注入门禁实现
### 3.1 新增 AST 写入路径检测
在 `check-style-profit-contracts.mjs` 中新增运行时动态注入分析：
- `ElementAccessExpression` 左值赋值检测（`=` / `??=` / `||=` / `&&=` / `+=` 等复合赋值）
- `Object.defineProperty(...)` 第二参数 key 检测
- `Object.defineProperties(...)` descriptor object 内 dynamic/unknown key 检测
- `Reflect.set(...)` 第二参数 key 检测
- `Object.assign(...)` source object literal 内 dynamic/unknown key 检测

### 3.2 fail closed 口径
在 style-profit surface 内，只要命中运行时动态属性注入路径且 key 非字符串字面量，统一失败：
- `style-profit forbids runtime dynamic property injection; use explicit literal keys（款式利润前端禁止运行时动态属性注入，请使用显式字面量键）`

### 3.3 读取型动态 key 边界
本任务未将纯读取型 dynamic key 作为阻断目标：
- `const value = item[ACTION_KEY]`
- `if (item[ACTION_KEY]) {}`

前提：该读取不构成写入、定义、设置、合并或 action 入口构造。

### 3.4 依赖边界
- 未新增依赖。
- 继续使用现有 `typescript` devDependency。
