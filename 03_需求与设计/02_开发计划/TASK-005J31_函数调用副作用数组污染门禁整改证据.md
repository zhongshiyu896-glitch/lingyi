## 2. 核心实现说明

### 2.1 函数副作用摘要模型落地

在 `collectRuntimeAnalysisContext(...)` 内新增函数副作用摘要体系：

- `runtimeFunctionSummaryMap`
- `runtimeFunctionNodeSummaryMap`
- `analyzeRuntimeFunctionSummary(...)`
- `resolveRuntimeFunctionSummaryFromExpression(...)`
- `applyRuntimeFunctionSummaryAtCall(...)`

摘要字段覆盖：

- `function_id`
- `declared_name`
- `aliases`
- `captures_tracked_arrays`
- `mutates_array_ids`
- `escapes_array_ids`
- `has_unknown_side_effect`
- `summary_confidence`

并支持 FunctionDeclaration 预收集（hoist 语义）后再进入主遍历。

### 2.2 数组污染从“函数体位置”改为“调用点生效”

通过 `applyRuntimeFunctionSummaryAtCall(...)` 在 `CallExpression` 调用点应用污染：

1. `mutates_array_ids` -> `tainted`
2. `escapes_array_ids` -> `escaped`
3. `has_unknown_side_effect/summary_confidence=unknown` -> `unknown`

实现后，`poison()` 在 `new Worker(...args)` 之前调用时，污染会准确作用于使用点，不再受函数体源码位置影响。

### 2.3 函数别名与等价调用接线

新增接线点：

1. 变量声明/赋值时调用 `setRuntimeFunctionAliasBinding(...)` 建立函数别名映射。
2. `resolveRuntimeFunctionSummaryFromExpression(...)` 新增 `call/apply/bind` 反解：
   - `poison.call(...)`
   - `poison.apply(...)`
   - `poison.bind(...)`
3. 调用点无法解析到明确函数摘要，且为可疑无参函数调用时，对 tracked arrays 执行 conservative `unknown` 污染（style-profit surface fail closed）。

### 2.4 闭包/IIFE/函数容器场景覆盖

J31 新增用例已覆盖：

- 函数声明提升（hoisted FunctionDeclaration）
- 箭头函数与函数表达式
- 函数别名链调用
- `call/apply/bind` 调用
- 闭包返回函数与 IIFE 立即调用
- 无参未知函数调用 `maybePoison()`
- 函数数组容器 `fns[0]()`
- 与 `Reflect.construct(Worker, args)`、`new unknownCtor(...args)` 的联动污染
