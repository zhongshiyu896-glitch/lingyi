## 一、任务目标

修复 `check-style-profit-contracts.mjs` 对“函数返回 Worker/SharedWorker 构造器”的高危绕过。将 `new`、`Reflect.construct`、`bind`、函数返回、IIFE、条件返回统一收敛到同一套 constructor invocation descriptor，确保动态构造器与高危 URL 组合时 fail closed。

本任务只修前端只读契约门禁，不开放款式利润快照创建入口，不进入 TASK-006。

## 二、背景问题

TASK-005J26 已修复 `Worker.bind(...)`、绑定构造器别名和 `Reflect.construct(...)` 主路径，但审计官第 145 份审计复现以下绕过：

```ts
const getWorker = () => Worker
new (getWorker())('data:text/javascript,postMessage(1)', { type: 'module' })
```

根因是 `new` 路径和 `Reflect.construct` 路径对未知构造目标的兜底不一致。动态函数返回 Worker/SharedWorker 构造器后，未被归一到 Worker constructor sink，导致高危 URL 继续漏判。

## 四、实现要求

### 4.1 统一 constructor invocation descriptor

必须将以下入口统一到同一个 constructor invocation descriptor：

1. `new Worker(url, options)`
2. `new Alias(url, options)`
3. `new (Worker.bind(null))(url, options)`
4. `Reflect.construct(Worker, [url, options])`
5. `new (getWorker())(url, options)`
6. `Reflect.construct(getWorker(), [url, options])`
7. `new ((() => Worker)())(url, options)`
8. `new (condition ? Worker : Worker)(url, options)`

要求：

1. descriptor 必须包含 `ctor_kind`、`ctor_confidence`、`url_expression`、`options_expression`、`source_trace`。
2. `ctor_kind` 可为 `worker`、`shared_worker`、`safe_known_constructor`、`unknown`。
3. `ctor_confidence=unknown` 且 URL 为 data/blob/http/https/javascript/变量/模板/函数返回时，在 style-profit surface 内必须 fail closed。
4. 所有 Worker/SharedWorker descriptor 必须复用同一套 Worker URL 校验函数。
5. 不允许 `new` 和 `Reflect.construct` 分别维护两套兜底逻辑。

### 4.2 函数返回 Worker/SharedWorker 必须识别

必须识别并拦截以下形式：

```ts
const getWorker = () => Worker
new (getWorker())('data:text/javascript,postMessage(1)', { type: 'module' })

function getSharedWorker() { return SharedWorker }
new (getSharedWorker())('data:text/javascript,postMessage(1)')
```

要求：

1. 零参数函数直接返回 Worker/SharedWorker 时，返回值必须映射为对应构造器。
2. 函数返回 Worker/SharedWorker 别名时，必须继续解析别名。
3. 函数返回 `globalThis.Worker` / `window.SharedWorker` 时，必须识别为对应构造器。
4. 函数返回条件表达式时，两分支均为同类 Worker 构造器才可映射；否则 fail closed。
5. 函数体复杂、返回值多分支不可静态证明、返回 unknown constructor 时，在 style-profit surface 内 fail closed。

### 4.3 IIFE 返回构造器必须识别

必须识别并拦截以下形式：

```ts
new ((() => Worker)())('data:text/javascript,postMessage(1)', { type: 'module' })
new ((function () { return SharedWorker })())('data:text/javascript,postMessage(1)')
```

要求：

1. 箭头函数 IIFE 返回 Worker/SharedWorker 必须识别。
2. function IIFE 返回 Worker/SharedWorker 必须识别。
3. IIFE 返回无法静态解析的构造器时 fail closed。
4. IIFE 返回静态本地安全 worker 构造路径时仍需继续走 URL 校验，不得直接放行。

### 4.4 Reflect.construct 动态构造器必须识别

必须识别并拦截以下形式：

```ts
const getWorker = () => Worker
Reflect.construct(getWorker(), ['data:text/javascript,postMessage(1)', { type: 'module' }])

Reflect.construct((() => Worker)(), ['data:text/javascript,postMessage(1)'])
```

要求：

1. Reflect.construct 第一个参数为函数返回值时，必须走同一套 constructor resolver。
2. 第一个参数解析为 Worker/SharedWorker 时，第二个参数数组中的 URL 必须走 Worker URL 校验。
3. 第一个参数无法静态证明安全且第二个参数数组含高危 URL 或未知 URL 时 fail closed。
4. Reflect.construct 的 `newTarget` 不得豁免构造器解析失败。

### 4.5 未知构造目标兜底必须统一

要求：

1. `new unknownCtor(highRiskUrl)` 与 `Reflect.construct(unknownCtor, [highRiskUrl])` 必须同口径处理。
2. highRiskUrl 包括 data/blob/http/https/javascript 字符串、变量 URL、模板 URL、函数返回 URL、Blob URL 标记变量。
3. 在 style-profit surface 内，未知构造目标 + highRiskUrl 必须 fail closed。
4. 已知安全构造器如 `Date`、`URL`、`Error` 不应被误杀，但必须由明确白名单证明，不得用“未知即放行”。

## 五、必须新增反向测试

至少新增以下反向测试，每条必须断言门禁失败：

1. `const getWorker = () => Worker; new (getWorker())('data:text/javascript,postMessage(1)', { type: 'module' })`
2. `function getWorker() { return Worker }; new (getWorker())('data:text/javascript,postMessage(1)')`
3. `const getSharedWorker = () => SharedWorker; new (getSharedWorker())('data:text/javascript,postMessage(1)')`
4. `const getWorker = () => globalThis.Worker; new (getWorker())('data:text/javascript,postMessage(1)')`
5. `const W = Worker; const getWorker = () => W; new (getWorker())('data:text/javascript,postMessage(1)')`
6. `new ((() => Worker)())('data:text/javascript,postMessage(1)', { type: 'module' })`
7. `new ((function () { return SharedWorker })())('data:text/javascript,postMessage(1)')`
8. `const getWorker = () => condition ? Worker : Worker; new (getWorker())('data:text/javascript,postMessage(1)')`
9. `const getWorker = () => condition ? Worker : unknownCtor; new (getWorker())('data:text/javascript,postMessage(1)')` 必须 fail closed
10. `const getWorker = () => unknownCtor; new (getWorker())('data:text/javascript,postMessage(1)')` 必须 fail closed
11. `const getWorker = () => Worker; Reflect.construct(getWorker(), ['data:text/javascript,postMessage(1)', { type: 'module' }])`
12. `Reflect.construct((() => Worker)(), ['data:text/javascript,postMessage(1)'])`
13. `Reflect.construct(unknownCtor, ['data:text/javascript,postMessage(1)'])` 必须 fail closed
14. `new unknownCtor('data:text/javascript,postMessage(1)')` 必须 fail closed
15. `new unknownCtor(workerUrl)` 中 `workerUrl` 无法静态证明安全时 fail closed
16. `Reflect.construct(unknownCtor, [workerUrl])` 中 `workerUrl` 无法静态证明安全时 fail closed
