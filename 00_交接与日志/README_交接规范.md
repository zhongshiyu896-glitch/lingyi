# 交接规范（领意服装管理系统）

## 目标
确保任意账号切换后，下一位开发者可在 10 分钟内接手继续开发。

## 必填记录
1. 每次开发开始前，在 `WORK_LOG.md` 增加一条“开始记录”（时间、操作者、目标）。
2. 每次完成一个可验证动作（改代码/建表/改流程/跑测试），在 `WORK_LOG.md` 增加“结果记录”。
3. 每日结束前，更新 `HANDOVER_STATUS.md`：
   - 当前完成度
   - 阻塞点
   - 下一步
   - 关键命令

## 文件约定
1. 所有新增代码放在 `02_源码/`。
2. 所有环境与部署文件放在 `03_环境与部署/`。
3. 所有测试证据（截图、JSON、报告）放在 `04_测试与验收/`。
4. 所有阶段交付包放在 `05_交付物/`。

## 日志风格
1. 结论先行：先写“做了什么/结果是什么”。
2. 命令可复现：记录关键命令、输入、输出路径。
3. 可追溯：每条记录包含绝对路径与时间。
